# AI disclosure

## Commits and pull requests

Include a `Co-Authored-By` trailer in any commit where Claude wrote or substantially
rewrote code:

```
Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

Add a one-line note to the PR description when Claude contributed significant design
or implementation:

```
🤖 Drafted with Claude Code
```

Omit both for trivial edits — typo fixes, comment rewording, and formatting-only
changes do not require attribution.

## Code comments

NEVER annotate lines as "AI-generated" inline. The commit trailer is the right place
for attribution. Inline "AI-generated" comments create noise, rot as code evolves,
and obscure the actual intent of the comment.

Only add a comment when the WHY is non-obvious regardless of origin: a hidden
constraint, a subtle invariant, or a workaround for a specific bug. If removing the
comment would not confuse a future reader, do not write it.

## Public documentation

**Changelog entries** — write in the project's voice. Do not reference Claude or the
session that produced the change. The changelog records what changed and why, not how
it was written.

**README** — do not add an "AI-assisted" badge unless the project owner explicitly
decides to. That is a project-level policy decision, not a per-commit one.

**Release notes** — auto-generated by `release.yml` from merged PRs and commits. No
AI attribution needed in release notes.
